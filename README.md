#Productivity Time Tracker – Chrome Extension
Project Overview

Productivity Time Tracker is a browser-based productivity monitoring application developed as a Chrome Extension to help users understand and improve their online browsing habits. The extension automatically tracks the amount of time spent on different websites and categorizes them into Productive, Unproductive, and Neutral groups. The collected browsing information is synchronized with a backend server and displayed through an interactive dashboard that provides meaningful productivity insights.

The project demonstrates the practical implementation of browser extension development, real-time website activity tracking, REST API communication, and dashboard visualization. It provides users with a simple and effective way to analyze how their online time is distributed and helps them make better decisions regarding productivity and time management.

The complete application was developed with the assistance of Cursor AI, which accelerated the coding process by generating the project structure, backend logic, extension components, and dashboard interface. The generated code was configured, tested, and refined to build a functional productivity tracking system.

Project Objective

The primary objective of this project is to develop a browser extension capable of automatically monitoring website usage, collecting browsing statistics, classifying websites according to productivity levels, and presenting meaningful analytics through an interactive dashboard. The project also aims to demonstrate the integration of Chrome Extension APIs with a Node.js backend while providing users with real-time productivity insights.

Working Process

The Productivity Time Tracker operates as a Chrome Extension that continuously monitors the active browser tab while the user browses the internet. The extension records the duration spent on each website and compares the visited domain with predefined productivity categories. Based on this classification, the browsing activity is labeled as productive, unproductive, or neutral.

The tracked browsing information is periodically synchronized with a backend server developed using Node.js and Express.js. The backend processes the collected data and stores it in a local JSON database. The dashboard retrieves this processed information through REST APIs and displays productivity statistics, total browsing time, productivity score, category-wise time distribution, and the most frequently visited websites. This workflow enables users to monitor their browsing habits and evaluate their online productivity through a clear and organized interface.
